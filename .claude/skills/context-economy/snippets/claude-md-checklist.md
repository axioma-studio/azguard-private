# Чеклист аудита CLAUDE.md

Прогоняйте при каждом заметном росте файла. Ориентир Anthropic: **≤200 строк**.

## Размер

| Строк | Оценка |
|---|---|
| >500 | критически много — success rate падает, стоимость +20% |
| 200–500 | нужен trim |
| <200 | официальный ориентир |
| <100 | цель |

## По каждой строке

- [ ] Удивит ли это опытного разработчика, нового в репо? Нет → удалить.
- [ ] Это знает модель из обучения (стандартный фреймворк/язык)? Да → удалить.
- [ ] Это aspirational («пиши чистый код», «будь внимателен»)? → удалить.
- [ ] Это контакты/расписания/FAQ без действия? → удалить.
- [ ] Это многошаговая процедура? → вынести в скилл.
- [ ] Это нужно только для части кодовой базы? → вынести в `.claude/rules/<topic>.md` с `paths:`.
- [ ] Это заметка для людей, не для агента? → обернуть в `<!-- ... -->` (вырезается до инжекции).

## Анти-паттерны

- `@import` ради «экономии» — импортированные файлы грузятся при старте целиком;
  это организация текста, токены не экономит.
- Два правила противоречат друг другу — модель выберет произвольно; устранить конфликт.
- Дубли между CLAUDE.md, вложенными CLAUDE.md и `.claude/rules/` — оставить одно место.
