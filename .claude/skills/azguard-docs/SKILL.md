# azguard-docs

Use when writing or updating README, VitePress documentation, CHANGELOG, or contributing guides.

## Documentation Locations

- `README.md` — root-level package overview and quick-start
- `docs/` — VitePress site (English + Russian), deployed to GitHub Pages via `docs.yml`
- `CHANGELOG.md` — auto-generated by git-cliff on release tags
- `.github/CONTRIBUTING.md` — contribution workflow

## VitePress

- Dev server: `npm run docs:dev`
- Build: `npm run docs:build`
- Preview: `npm run docs:preview`

## README Guidelines

- Keep installation and quick-start accurate to the current API.
- Document all three packages (core, filament, context) separately.
- Code examples must be runnable — test them against the current source.

## CHANGELOG

CHANGELOG is generated automatically by git-cliff on version tags — do not edit it manually. Commit messages must follow Conventional Commits for correct categorization:

| Prefix | Section |
|---|---|
| `feat` | Features |
| `fix` | Bug Fixes |
| `docs` | Documentation |
| `refactor` | Refactoring |
| `perf` | Performance |
| `test` | Tests |
| `ci` | CI/CD |
| `chore` | Maintenance |

## Do Not

- Edit `CHANGELOG.md` by hand.
- Add docs for unreleased or hypothetical features.
- Skip updating README when the public API changes.
